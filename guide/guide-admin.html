1. 어드민 권한과 책임

어드민: Member05 · Member04 · Member27 — 이 3인만 관리 패널 접근 가능.

1-1. 권한 목록

  • 일정(타임테이블) 추가·수정·삭제
  • 각 사람별 할일 등록·수정·삭제
  • 공지사항 등록·삭제 (전체 사용자에게 노출)
  • 시뮬레이션 시간 설정 (모든 기기에 동기화됨)
  • 전체 일정 보기에서 모든 사람의 담당 표시

1-2. 책임

⚠ 모든 변경은 즉시 모든 기기에 반영됩니다
편집 전 반드시 의도한 변경인지 확인. 잘못 삭제한 경우 재입력 필요 (롤백 기능 없음).

2. 관리 패널 진입

  1. 본인 어드민 계정으로 로그인합니다.
  2. 탑바 우측에 관리 버튼이 표시되어 있는지 확인.
  3. 관리 버튼 클릭 → 관리 패널 오버레이 진입.
📌 비어드민 계정에서는 관리 버튼이 보이지 않음
CSS와 JS 양쪽에서 권한 체크가 들어가 있어, 비어드민이 우회 접근하는 것을 막습니다.

3. 일정 관리 (타임테이블 편집)

3-1. 시간대 추가

  1. 1일차 / 2일차 탭 선택.
  2. 리스트 하단 + 시간대 추가 클릭.
  3. 시간(HH:MM), 일정명 입력.
  4. 야간 일정이면 야간 체크박스 켜기 (24시 이후 표시 처리에 영향).

3-2. 담당자 추가/수정

  1. 해당 시간대의 업무 편집 버튼 클릭.
  2. + 담당자 추가하기 또는 기존 항목 클릭.
  3. 이름은 자동완성 데이터리스트에서 선택 (오타 방지).
  4. 직책(role)은 자동 매핑되니 수정 불필요.
  5. 업무 내용 입력.

3-3. 시간대 순서 변경

각 시간대 우측의 ▲▼ 버튼으로 순서 조정. 시간(HH:MM) 자체는 별도 입력해야 함.

3-4. 시간대 / 담당자 삭제

휴지통 🗑 아이콘 클릭. 확인 모달 없이 즉시 삭제되니 신중히.

⚠ 담당자 0명 시간대는 경고 표시
상단에 ⚠ N개 시간대 담당자 없음 배너로 알림. 의도된 경우(이동 시간 등)는 무시 가능.

4. 할일 관리

4-1. 사람별 할일 등록

  1. 관리 패널 → 할일 탭.
  2. 좌측 사람 목록에서 대상 선택.
  3. 우측 편집기에서 할일 항목 추가/수정/삭제.

4-2. 사용자가 보는 위치

할일 체크리스트는 메인 대시보드에서 제거되었으나, 데이터는 어드민 측에서 관리·확인 가능.

5. 공지사항 작성

  1. 관리 패널 → 공지 탭.
  2. 하단 텍스트 영역에 공지 내용 입력.
  3. 공지 등록 버튼 클릭.
  4. 전체 사용자 대시보드 상단에 5초 내 노출.

공지 작성 권장 형식

  • 한 줄 요약 + 시간/장소 명시
  • 예: "19:00 강당 집결 — 리허설 시작 전 인원 점호"
  • 긴 내용은 카톡/문자로 보내고, 공지는 헤드라인만

6. 시뮬레이션 모드 (사전 테스트)

실제 시간이 아니라 가상 시간으로 앱 동작을 미리 확인할 수 있습니다.

6-1. 사용 절차

  1. 관리 패널 → 일정 탭.
  2. 상단 시뮬레이션 모드 토글 ON.
  3. 날짜·시간 선택 (예: 2026-05-08 16:30).
  4. 대시보드/타임테이블이 해당 시각 기준으로 업데이트됨.

6-2. 중요

🚨 시뮬레이션은 모든 기기에 동기화됩니다
본인이 ON 했을 때 다른 사용자도 시뮬레이션 시간을 보게 됩니다.
행사 당일에는 반드시 OFF 상태로 두세요.

6-3. 종료

토글을 OFF로 변경하면 즉시 실제 시간으로 복귀.

7. 비밀번호 초기화

사용자가 비밀번호를 잊었을 때 어드민이 직접 초기화 가능.

7-1. 방법

현재 GUI에는 직접적인 초기화 버튼이 없습니다. AWS 콘솔 또는 다음 절차로 초기화합니다.

  1. 기획국 Member05(010-XXXX-XXXX)에게 초기화 요청 전달.
  2. 관리자가 DynamoDB에서 해당 사용자의 비밀번호 항목 삭제.
  3. 다음 로그인부터 기본 비밀번호 abc123! 사용 가능.

7-2. 비밀번호 저장 방식

  • SHA-256 + 솔트(이름) 해시로 저장
  • 서버에는 평문 비밀번호가 저장되지 않음
  • 해시 충돌 우려 없음 (실질적으로 0%)

8. 시스템 아키텍처

8-1. 구성도

[브라우저]
   │ 5초 폴링 (GET) / 변경 시 PUT
   ▼
[API Gateway HTTP API]
   YOUR-API-ID.execute-api.YOUR-REGION.amazonaws.com
   │
   ▼
[Lambda: YOUR-LAMBDA-NAME]
   Node.js 20.x · 128MB · 10s timeout
   │
   ▼
[DynamoDB: YOUR-TABLE-NAME]
   파티션 키: dataKey
   PAY_PER_REQUEST

[S3: YOUR-S3-BUCKET]
   정적 호스팅 (HTTPS)

8-2. 동기화 키

키내용쓰기 권한
schedule1일차/2일차 일정·담당자어드민
notices공지사항 배열어드민
todos사람별 할일어드민
passwords비밀번호 해시 맵각 사용자 본인
sim시뮬레이션 시간 상태어드민

8-3. 비용

  • DynamoDB on-demand: 행사 당일 약 $1 미만
  • Lambda · API Gateway: 무료 티어 내
  • S3: 무시할 수준

9. 장애 대응 / 문의

9-1. 자주 발생하는 이슈

Q. 사용자가 "비밀번호가 맞지 않습니다" 호소.

  • 1-2초 후 재시도 안내 (서버 동기화 대기)
  • 그래도 실패 → 브라우저 캐시 삭제 + 새로고침
  • 위 모두 실패 → 비밀번호 초기화 (7장 참조)

Q. 일정 변경이 다른 기기에 안 보임.

  • 해당 기기 새로고침 (5초 내 자동 반영되어야 정상)
  • 탑바에 "오프라인" 표시면 인터넷 확인

Q. 어드민이 관리 버튼이 안 보임.

  • 본인 계정이 ADMIN_SET에 포함되어 있는지 확인 (Member05 / Member04 / Member27)
  • 로그아웃 → 재로그인

Q. 시뮬레이션 모드 끈 후에도 다른 기기는 시뮬 시간으로 보임.

  • 최대 5초 대기 (다음 폴링까지)
  • 그래도 안 풀리면 새로고침

9-2. 데이터 백업

DynamoDB는 자동 백업 미설정. 행사 직전 수동 export 권장:

aws dynamodb scan --table-name YOUR-TABLE-NAME \
  --region ap-northeast-2 --output json > backup.json
